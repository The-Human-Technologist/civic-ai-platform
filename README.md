# Civic AI & Road Safety Intelligence Platform

[![CI](https://github.com/The-Human-Technologist/civic-ai-platform/actions/workflows/ci.yml/badge.svg)](https://github.com/The-Human-Technologist/civic-ai-platform/actions/workflows/ci.yml)
[![License: AGPL-3.0](https://img.shields.io/badge/License-AGPL--3.0-blue.svg)](LICENSE)
[![Hosted prototype](https://img.shields.io/badge/Hosted_prototype-open-22c55e?logo=vercel)](https://civic-ai-platform-three.vercel.app)

Privacy-first, open-source civic video intelligence for municipalities, road-safety teams, and
authorized pilot operators. The working prototype processes uploaded video with a sequential
multi-model pipeline, converts model signals into advisory events, and requires a human decision
before anything leaves review.

## Current release

| Capability | Status |
|---|---|
| Authorized MP4/WebM/MOV/AVI upload | Working |
| Real YOLO26n inference | Working in the local FastAPI worker |
| NVIDIA CUDA / FP16 | Automatic when PyTorch detects the GPU |
| Person and road-vehicle detection | Working with standard weights |
| Congestion advisory | Working from person/vehicle density thresholds |
| Pothole, road-damage, and garbage advisory | Working with the optional Indian-road specialist checkpoint |
| Waterlogging advisory | Working experimentally with YOLO-World; field verification mandatory |
| Helmet non-compliance advisory | Working experimentally with a specialist checkpoint |
| Wrong-way movement advisory | Working with YOLO26 + ByteTrack and per-camera direction input |
| Privacy-safe evidence image | Whole people and vehicles are masked before return |
| Human review queue | Working: confirm, reject, or request field verification |
| Reports, dashboards, and pilot proposal | Working |
| Synthetic workflow samples | Available and explicitly labeled |
| Live CCTV / RTSP | Not enabled |
| Facial recognition / citizen scoring | Prohibited |
| Automatic challan or enforcement | Prohibited |

The system deliberately labels helmet and waterlogging modules experimental. All specialist
checkpoints must be evaluated on the proposed camera angles and local conditions before an
operational pilot. The prototype does not claim crash detection or legal speed measurement.

## Why YOLO26n

The default is the nano YOLO26 checkpoint at 640px. It is the practical balance for the target
MSI Thin 15 laptop with a 4GB RTX 3050 and 16GB system RAM. CUDA FP16 is used automatically; CPU
fallback remains available but is slower. TensorRT export is an optional later optimization after
the PyTorch path is verified on the actual machine.

## Safety contract

- authorized or properly licensed clips only
- authorization reference required by both the Next.js API and Python worker
- no URL downloads, scraping, public CCTV discovery, or live stream intake
- no facial recognition, biometric identification, or citizen scoring
- no automatic fine, challan, prosecution, or field dispatch
- all alerts start at `pending` human review
- the worker deletes the uploaded clip and decoded frames after inference
- only privacy-masked evidence images and small detection metadata return to the app
- raw footage and model weights are gitignored

See [PRIVACY.md](PRIVACY.md), [SECURITY.md](SECURITY.md), and
[DATA_SOURCES.md](DATA_SOURCES.md).

## Architecture

```text
Browser
  -> Next.js multipart API (validates file, size, location, authorization)
  -> local FastAPI worker
  -> OpenCV frame sampling
  -> sequential YOLO26n, civic, helmet, YOLO-World, and ByteTrack modules
  -> whole-person / whole-vehicle privacy masking
  -> advisory detections + masked evidence
  -> human review queue
```

Heavy inference never runs in a Vercel route. For the police walkthrough, run the Next.js app and
AI worker on the RTX 3050 laptop.

## Quick start

### 1. Web app

```powershell
npm ci
Copy-Item .env.example .env.local
npm run dev
```

### 2. AI worker

Use Python 3.11 or 3.12. From `services/ai-worker`:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
Copy-Item .env.example .env
python scripts\bootstrap_model.py
python scripts\doctor.py
uvicorn main:app --host 127.0.0.1 --port 8000
```

If the doctor says `CUDA available: False`, install the Windows CUDA build of PyTorch using the
command generated by the official PyTorch installer page, then run the doctor again.

### 3. Run a real analysis

1. Open `http://localhost:3000/dashboard/upload`.
2. Check worker health; confirm the model and GPU are ready.
3. Select an authorized or licensed clip.
4. Enter the real location and authorization/licence reference.
5. Confirm authorization.
6. Select the analysis modules and, for wrong-way analysis, the permitted camera direction.
7. Run multi-model analysis.
8. Open the event queue and review privacy-masked evidence.

## Configuration

Web (`.env.local`):

```dotenv
AI_PROCESSING_MODE=worker
AI_WORKER_URL=http://localhost:8000
MAX_VIDEO_UPLOAD_MB=250
```

Worker (`services/ai-worker/.env`):

```dotenv
YOLO_MODEL_PATH=models/yolo26n.pt
YOLO_DEVICE=auto
YOLO_IMAGE_SIZE=640
YOLO_CONFIDENCE=0.35
CIVIC_MODEL_PATH=models/civic-road-issues.pt
HELMET_MODEL_PATH=models/helmet-candidate.pt
WATERLOGGING_MODEL_PATH=models/waterlogging-world.pt
FRAME_SAMPLE_RATE_FPS=1
MAX_SAMPLED_FRAMES=300
PRIVACY_MASKING_ENABLED=true
ALLOW_MODEL_DOWNLOAD=false
```

Run `python scripts/bootstrap_model.py` to install all four ignored local checkpoints. Model
provenance, hashes, licences, and operational limitations are documented in
[docs/MODEL_PROVENANCE.md](docs/MODEL_PROVENANCE.md).

## Deployment boundary

This is a working **single-operator prototype**, not a completed statewide surveillance or
enforcement platform. A multi-user operational deployment still needs authenticated accounts,
RBAC, rate limiting, durable audit logs, an approved retention schedule, encrypted private object
storage, evaluated local civic datasets, legal review, monitoring, backups, and documented incident
response. Those controls are tracked in [ROADMAP.md](ROADMAP.md).
